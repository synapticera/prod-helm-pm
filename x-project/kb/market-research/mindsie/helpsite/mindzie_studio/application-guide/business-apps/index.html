Business Apps

Application Guide

Overview

Business Apps in mindzie Studio enable you to create simplified, user-friendly interfaces for business users to access process insights. Apps allow you to publish curated collections of dashboards and analysis, making process intelligence accessible to users who need to monitor and manage processes without navigating the full mindzie Studio environment.

What are Business Apps?

Business Apps provide:

  • Simplified Interface - Streamlined view focused on specific insights
  • Curated Content - Collections of relevant dashboards and analysis
  • Business User Access - Easy-to-use interface for non-technical users
  • Process Monitoring - Ongoing visibility into process performance
  • Insight Distribution - Push process intelligence down into the business

Purpose and Use Cases

Distributing Process Insights

  • Share analysis results with stakeholders
  • Provide ongoing process monitoring
  • Enable self-service data access
  • Reduce analyst workload

Business User Empowerment

  • Monitor processes without technical training
  • Access relevant metrics and KPIs
  • Track process improvements
  • Ensure compliance and SLA performance

Focused Views

  • Department-specific dashboards
  • Role-based access to insights
  • Process-specific monitoring
  • Custom business unit views

Accessing Business Apps

To access Business Apps:

  1. Click the Apps button in the top left corner
  2. Enter the Apps view

The Apps view displays:

  • Card layout of available apps
  • App thumbnails for easy identification
  • Management options for each app
  • Quick access to app content

Creating a New App

Step 1: Create App Framework

  1. Click New App in the top right corner
  2. Provide an App Name
  3. Configure Categories (optional)

Understanding Categories

Categories organize content within your app:

  • Optional - Not required for simple apps
  • Organization - Group related dashboards and analysis
  • Navigation - Help users find relevant content
  • Flexibility - Add as many or as few as needed

App Framework Created

Once saved, you have:

  • Empty app ready for content
  • Ability to assign users
  • Framework for adding dashboards and analysis

Managing User Access

Assigning Users to Apps

Similar to Projects, you can assign users with specific permissions:

  1. Click on the app
  2. Access user management
  3. Add users to the app

Permission Levels

Owner

  • Full control over app configuration
  • Can add/remove content
  • Can manage user access
  • Can edit and delete app

Contributor

  • Can access and view app content
  • Cannot modify app structure
  • Cannot manage users
  • Read-only access to insights

Security and Access Control

Apps provide an additional security layer:

  • Only assigned users can access apps
  • Control who sees specific insights
  • Separate access from full Studio permissions
  • Enable role-based content distribution

Adding Content to Apps

Business Apps can contain:

  • Dashboards Only - Visual metrics and KPIs
  • Analysis Only - Detailed process analysis views
  • Both Dashboards and Analysis - Comprehensive collections
  • Any Combination - Mix and match as needed

Adding Dashboards to Apps

From within mindzie Studio:

  1. Navigate to the dashboard you want to share
  2. Click on the dashboard
  3. Click the three dots menu
  4. Select Add to App
  5. Choose your app from the list
  6. Select the category (if configured)
  7. Or check Add to Top Level to add to main section

The dashboard is now published to the app.

Adding Analysis to Apps

From the analysis screen:

  1. Navigate to the analysis you want to share
  2. Click on the analysis
  3. Click the three dots menu
  4. Select Add to App
  5. Choose your app
  6. Select category or top level
  7. Click Add

The analysis is now available in the app.

Content Organization

Top Level Content

  • Appears in the main app view
  • Immediate access without navigation
  • Best for most important content
  • Simple, flat structure

Category Organization

  • Content grouped by category
  • Hierarchical navigation
  • Better for large app libraries
  • Organized by theme or department

Using Business Apps

Simplified Business User Interface

When users access an app:

  • Clean, simplified interface
  • Only shows published content
  • Easy navigation
  • Focused on insights, not configuration

User Experience

Business users can:

  • View Dashboards - Access visual metrics and KPIs
  • Explore Analysis - Interact with published analysis
  • Monitor Processes - Track ongoing performance
  • Access Insights - Self-service data exploration

Interaction Capabilities

Within the app, users can:

  • Interact with data
  • Monitor metrics and insights
  • Filter and explore published content
  • Access configured visualizations

AI Copilot Integration

Apps can be configured to include the mindzie AI Copilot:

AI Copilot in Apps

When enabled, users can:

  • Ask questions about the insights
  • Get AI-generated explanations
  • Explore data conversationally
  • Receive process recommendations

Benefits

  • Enhanced self-service capabilities
  • Reduced support burden on analysts
  • Deeper insight understanding
  • Natural language interaction

Sharing Apps

Distribution Method

Share apps simply by:

  1. Copy the app link - Get the URL from the browser
  2. Share the link - Email or message to business users
  3. Users access - Assigned users can open and use the app

Access Requirements

Users must:

  • Be assigned to the app
  • Have appropriate permissions
  • Have access to mindzie Studio environment

App Management

Managing Existing Apps

From the Apps card view, click the three dots on any app to:

Edit App

  • Modify app name and description
  • Update categories
  • Change app configuration

Adjust User Access

  • Add or remove users
  • Change permission levels
  • Control who can access the app

Add Thumbnails

  • Upload custom app images
  • Visual identification
  • Branding and organization

Delete App

  • Remove app completely
  • Does not delete underlying dashboards or analysis
  • Only removes the app container

Use Cases and Best Practices

Common App Types

Department-Specific Apps

  • Finance App - Financial process metrics
  • Operations App - Operational performance
  • Compliance App - Regulatory monitoring
  • Executive App - High-level KPIs

Process-Specific Apps

  • Order-to-Cash Monitoring - End-to-end process tracking
  • Procure-to-Pay Insights - Procurement analytics
  • Customer Service Metrics - Service performance
  • Manufacturing Operations - Production monitoring

Role-Based Apps

  • Manager Dashboard - Team performance metrics
  • Analyst View - Detailed process analysis
  • Executive Summary - High-level insights
  • Auditor Access - Compliance verification

Best Practices

Content Curation

  1. Relevance - Only include content relevant to app users
  2. Simplicity - Don't overwhelm with too many items
  3. Focus - Keep apps targeted to specific needs
  4. Updates - Regularly review and update content

Organization

  1. Clear Names - Use descriptive app names
  2. Categories - Organize large content libraries
  3. Thumbnails - Add visual identification
  4. Documentation - Explain app purpose to users

Access Management

  1. Appropriate Permissions - Give users the right level
  2. Regular Review - Audit user access periodically
  3. Security - Follow least-privilege principles
  4. Communication - Inform users when access changes

Performance

  1. Ensure Compliance - Use apps to monitor SLA performance
  2. Track Improvements - Show process improvement results
  3. Maintain Processes - Ongoing monitoring prevents regression
  4. Enable Action - Connect insights to business decisions
