‚úÖ Best Practice: Morning Brief Design

  • Push, don't pull: Executives shouldn't have to log in and navigate‚Äîdeliver insights proactively
  • Delta focus: Show what changed, not just current state ("‚Üë +2 from yesterday")
  • Action-oriented: Every insight includes a specific recommended action with context
  • 5-7 metrics max: Research shows executives are overwhelmed by more
  • Time-sensitive prioritization: Lead with what requires action TODAY

‚úÖ Best Practice: Health Score Display

  • Show contributing factors: Don't just show a number‚Äîexplain what drove it
  • Trend visualization: Include historical trajectory (improving/declining/stable)
  • Color coding with context: Red/Yellow/Green must include explanation
  • Component breakdown: Let executives see which measures are dragging down the score

‚ùå Anti-Pattern: Dashboard Failure Modes

From research: "Executives ask for spreadsheets because dashboards don't answer 'what should I do?'" Common failures include:

  • Showing raw metrics without context ("100 tickets" vs "95% met 4-hour SLA")
  • No escalation path when viewing a red metric
  • Stale data from infrequent updates (CS needs daily minimum)
  • Too many KPIs‚Äîresearch suggests 5-7 max for executive views

‚úÖ Best Practice: Account Prioritization

  • Revenue-weighted risk: Sort by ARR at risk, not just count of red accounts
  • Time-sensitive context: Surface renewal dates, SLA deadlines, stale accounts
  • Root cause visibility: Show WHY an account is at risk, not just that it is
  • Direct action links: One-click path to take the recommended action
  • Confidence intervals: ChurnZero's approach of showing prediction confidence
